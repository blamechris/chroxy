A run-it-yourself guide to capture the two empirical keystroke recordings that close #4882 (all-single-select multi-question form) and #4880 (≥10-option arrow-nav). You drive a live claude TUI; the recorder writes a JSONL to your temp dir ($TMPDIR on macOS, /tmp on Linux). This guide assumes you're running it alongside a chat session with an assistant (e.g. Claude Code) that can read that file from disk when you say "run 1 done" / "run 2 done" — that's a convenience of that workflow, not an automated feature of this repo. Working solo, just open the JSONL yourself (the recorder prints its path on exit).

0 · One-time setup do this once, in a normal Terminal tab

Open a real Terminal tab (Terminal.app / iTerm) — not inside this chat. The recorder needs raw keyboard access to capture arrow keys and Tab. 
cd /path/to/chroxy   # your chroxy checkout

The recorder spawns claude (from $CLAUDE_BIN, else ~/.local/bin/claude) in a PTY and logs every keystroke (in) and every screen byte (out). You'll run it twice — once per issue. Each run makes a fresh JSONL.

1 · Run 1 — #4882 all-single-select RUN 1 · this one definitely works

Goal: capture the bytes for a two-question, all single-select form — the smallest shape that wedged before the #4867 fix. We pick option 2 on both so the last answer digit (2) is distinct from the Submit screen's 1.

Launch it: 
node scripts/tui-form-recorder.mjs
When the claude prompt renders, paste: 
Use your AskUserQuestion tool to ask me exactly two single-select
questions before doing anything else: (1) pick a framework — React,
Vue, or Svelte; (2) pick a package manager — npm, pnpm, or yarn.
Ask both in one form, don't assume answers.
After Ctrl+D the recorder prints Recording: <tmpdir>/tui-form-recording-<ts>.jsonl. If an assistant is reading along, just say "run 1 done" and it opens the newest recording; solo, open that path yourself.

2 · Run 2 — #4880 ≥10-option attempt RUN 2 · either outcome is useful

Goal: get claude to render a single question with 10+ options and pick option 11 with the Down-arrow key. Heads-up: the AskUserQuestion tool schema caps at 4 options per question — so claude may refuse to list 12. That refusal is itself the finding (it means the arrow-nav path can't be reached via AskUserQuestion). Record whichever happens.

Launch it again (new recording): 
node scripts/tui-form-recorder.mjs
When the claude prompt renders, paste: 
Use your AskUserQuestion tool to ask me ONE single-select question
with twelve options to choose from (list 12 web frameworks).
Don't shorten the list.
If it caps at ~4 options: just pick any option, submit, Ctrl+D. Tell Claude "run 2 done — capped at N". Claude will document the path as defensive/dead and close #4880 on that basis — no goose chase. If it shows 10+: arrow ↓ down to option 11, Enter, let it submit, Ctrl+D, say "run 2 done — got 10+".
